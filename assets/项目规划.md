# AI Pulse — 项目规划（完善版）

> 基于「橘鸦AI早报」参考 + DailyDawn/Horizon/Agently 等开源项目调研
> 创建时间：2026-07-10
> 版本：v3.0

---

## 一、项目定位

**全自动 AI 日报/周报网站**——从采集到发布的全链路自动化，零人工干预。

- **核心**：信息源自动采集 + AI 智能整理 + 网站自动发布
- **原则**：全自动，零人工干预
- **暂不含视频**（后期扩展）
- **关键差异化**：结构化数据库存储 → 搜索/筛选/统计；现代 UI 交互 → 非静态页面

### 3.1 架构设计原则

1. **模块解耦**：采集→处理→生成→发布四层独立，任一层可单独替换
2. **数据驱动**：层与层之间通过结构化数据流转（Supabase），不依赖文件系统
3. **容错优先**：单个信息源失败不影响整体（借鉴DailyDawn的优雅降级）
4. **可观测性**：每层有日志和状态输出，生成结果写入 `generation_logs` 表

## 四、核心 Pipeline 详解

### 4.1 采集层（Collector）

**双通道采集策略**：

| 通道 | 技术 | 用途 | 调用方式 |
|------|------|------|----------|
| Web Search | coze-coding-dev-sdk | 发现昨日AI热点新闻 | API: `/api/daily/generate` |
| Fetch URL | coze-coding-dev-sdk | 提取新闻正文 | API: `/api/daily/generate` |

**采集流程**（借鉴 DailyDawn + Agently）：
```
1. Web Search 搜索 "AI news today 2026-07-09" + 中文 "AI新闻 2026年7月9日"
   → 获取候选 URL 列表（20-40条）
2. 并发 Fetch URL 提取每条新闻正文
   → 统一格式化: {title, content, url, source_name, published_at}
3. 输出原始资讯列表
```

**信息源分级**（由 Web Search 自然覆盖，非硬编码 RSS）：

| 级别 | 来源 | 覆盖内容 |
|------|------|----------|
| SSS | OpenAI Blog, Anthropic Blog, Google AI Blog | 官方重磅发布 |
| SS | Meta AI Blog, Microsoft AI Blog, DeepSeek, Hugging Face Blog | 一线动态 |
| S | 机器之心, 量子位, 新智元, TechCrunch AI, The Verge AI | 媒体报道 |
| A | arXiv cs.AI, GitHub Trending AI, Reddit r/MachineLearning | 学术/社区 |

### 4.2 处理层（Processor）

**处理 Pipeline**：
```
原始资讯 → 三层去重 → AI相关性判断 → AI摘要+分类 → 重要度打分 → 筛选 → 内容增强
```

**三层去重**（借鉴橘鸦 + Horizon）：
1. **URL去重**：相同链接直接跳过
2. **标题去重**：LLM 判断标题语义相似度 > 0.85 视为重复
3. **事件去重**：LLM 判断是否与已入库新闻描述同一事件，保留信息最完整的

**AI 相关性判断**（核心约束）：
- LLM 在处理每条新闻时，首先判断「是否与 AI 直接相关」
- 判断标准：内容的核心主题是否为人工智能技术/产品/政策/研究
- 非 AI 内容直接丢弃，不计入任何统计
- Prompt 中给出明确示例：✅ GPT-5发布 / ❌ 通用科技新闻

**自动分类**（融合橘鸦8分类 + 自身定位）：

| 编号 | 分类 | 说明 | 对应英文搜索关键词 |
|------|------|------|-------------------|
| 01 | 要闻 | 重大突破、重磅发布 | breakthrough, major release |
| 02 | 模型发布 | 新模型、模型更新 | model release, LLM update |
| 03 | 开发生态 | Agent、SDK、工具、框架 | agent, SDK, framework |
| 04 | 产品应用 | 产品功能、用户端更新 | product launch, feature update |
| 05 | 技术与洞察 | 技术解析、研究报告 | research, technical analysis |
| 06 | 行业动态 | 融资、上市、政策 | funding, IPO, policy |
| 07 | 开源项目 | 开源发布、重大更新 | open source, GitHub |
| 08 | 前瞻与传闻 | 传闻、预告、规划 | rumor, preview, roadmap |

**重要度打分**（借鉴 Horizon 的多维度评分）：

| 维度 | 说明 | 分值 |
|------|------|------|
| 影响力 | 影响多少人/行业 | 1-10 |
| 时效性 | 是否刚发生 | 1-10 |
| 独特性 | 是否独家/首次 | 1-10 |
| 可靠性 | 信源可信度 | 1-10 |

等级映射：
| 总分 | 等级 | 处理 |
|------|------|------|
| ≥35 | SSS | 置顶要闻 + 内容增强 |
| 25-34 | SS | 重要进展 |
| 15-24 | S | 值得关注 |
| 8-14 | A | 一般资讯 |
| <8 | B | 过滤掉 |

**内容增强**（借鉴 Horizon）：
- SSS 级新闻自动触发 Web Search 补充背景信息
- 在摘要中融入多源视角
- 关联相关历史新闻（从数据库检索）

### 4.3 生成层（Generator）

**日报生成**：
```
1. 从 Supabase 读取当天处理完成的新闻条目
2. 按分类排序（要闻→模型→开发生态→...），同分类按等级降序
3. LLM 为每个分类生成1-2句导语
4. LLM 生成当日概览（2-3句话概括今天最重要的事）
5. 输出写入 daily_report 表
```

**周报生成**：
```
1. 从 Supabase 读取上周所有 news_article
2. LLM 生成周度概览（3-5句话）
3. LLM 分析本周趋势洞察（技术趋势 + 行业趋势 + 投资热点）
4. 按分类精选本周 Top 条目
5. 输出写入 weekly_report 表
```

### 4.4 发布层（Publisher）

- **Supabase 入库**：所有结构化数据持久化
- **网站更新**：Next.js ISR（增量静态再生）或按需 revalidation
- **RSS 2.0 Feed**：`/api/rss` 输出标准 RSS

### 4.5 调度层（Scheduler）

| 任务 | 触发方式 | 时间 | 说明 |
|------|----------|------|------|
| 日报生成 | 外部 Cron / 手动 | 每日 07:00 UTC | 调用 `/api/daily/generate` |
| 周报生成 | 外部 Cron / 手动 | 每周一 08:00 UTC | 调用 `/api/weekly/generate` |
| 排行榜刷新 | 手动触发 | 按需 | 调用 `/api/leaderboard/refresh` |

## 五、数据库设计（Supabase）

### 5.1 news_article

```sql
CREATE TABLE news_article (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,                -- AI生成的中文标题
  summary TEXT NOT NULL,              -- AI摘要，200字内
  original_title TEXT,                -- 原文标题（可能为英文）
  source_url TEXT NOT NULL,           -- 原文链接
  source_name TEXT NOT NULL,          -- 来源名
  category TEXT NOT NULL,             -- 分类(01-08)
  importance TEXT NOT NULL,           -- 等级: SSS/SS/S/A
  importance_score INTEGER,           -- 原始分数
  published_at TIMESTAMPTZ NOT NULL,  -- 发布时间
  daily_date DATE NOT NULL,           -- 归属日报日期
  is_ai_related BOOLEAN DEFAULT TRUE, -- AI相关性（默认true，非AI已过滤）
  keywords TEXT[],                    -- AI提取的关键词
  background_info TEXT,               -- 内容增强（仅SSS级）
  related_ids UUID[],                 -- 关联新闻ID
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_news_daily_date ON news_article(daily_date DESC);
CREATE INDEX idx_news_category ON news_article(category);
CREATE INDEX idx_news_importance ON news_article(importance);
CREATE INDEX idx_news_source_url ON news_article(source_url);  -- 去重用
```

### 5.2 daily_report

```sql
CREATE TABLE daily_report (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  report_date DATE NOT NULL UNIQUE,   -- 日报日期
  overview TEXT NOT NULL,             -- AI生成的当日概览
  category_introductions JSONB,       -- 各分类导语 {category: intro}
  article_count INTEGER DEFAULT 0,    -- 收录条数
  generated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 5.3 weekly_report

```sql
CREATE TABLE weekly_report (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  week_start DATE NOT NULL UNIQUE,    -- 周一日期
  week_end DATE NOT NULL,             -- 周日日期
  summary TEXT NOT NULL,              -- 周度概览
  trends TEXT NOT NULL,               -- 趋势洞察
  highlights JSONB NOT NULL,          -- 本周精选 [{title, summary, category, importance}]
  article_count INTEGER DEFAULT 0,
  generated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 5.4 llm_model

```sql
CREATE TABLE llm_model (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  provider TEXT NOT NULL,
  category TEXT NOT NULL,             -- 闭源/开源/本地
  params_count TEXT,                  -- 参数量
  release_date DATE,
  scores JSONB NOT NULL,              -- {mmlu, humaneval, arena_elo, mt_bench, math, mmmu}
  logo_url TEXT,
  trend TEXT DEFAULT '→',             -- ↑↓→
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 5.5 generation_log（可观测性）

```sql
CREATE TABLE generation_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT NOT NULL,                 -- 'daily' | 'weekly' | 'leaderboard'
  target_date DATE,                   -- 目标日期
  status TEXT NOT NULL,               -- 'running' | 'success' | 'failed'
  articles_found INTEGER DEFAULT 0,
  articles_after_dedup INTEGER DEFAULT 0,
  articles_after_filter INTEGER DEFAULT 0,
  error_message TEXT,
  started_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);
```

## 九、与现有前端代码的对接

当前前端已有3个页面（首页/排行榜/详情），使用 Mock 数据。后端开发完成后需要：

1. **替换 Mock**：将 `mock-data.ts` 的导入替换为 API 调用
2. **添加 loading 状态**：骨架屏、空态、错误态
3. **日期切换**：首页增加日期选择器，调用 `/api/daily?date=xxx`
4. **ISR/SSR**：页面改为服务端渲染，提升 SEO 和首屏速度
